# Claude Design プロンプト — 植物マスタ管理画面

農園モニタリングアプリ「ミニ農園」の植物マスタ管理画面（/settings/plants）をデザインしてください。

---

## アプリ概要

ESP32センサーデータをリアルタイム監視する農園ダッシュボード。
技術スタック: Next.js + shadcn/ui + Tailwind CSS + lucide-react アイコン。

---

## 共通レイアウト

**ヘッダー（全画面共通）**
- 左端: アプリ名「ミニ農園」 + 緑の点滅ドット「更新中」
- 中央: ナビゲーション「ダッシュボード」「ゾーン」「アラート」「植物マスタ」（現在のページはアクティブ状態）
- 右端: ベルアイコン（アラート件数バッジ付き）+ ユーザーアバター

---

## デザインシステム

**カラートークン（Tailwind セマンティック）**
- `text-content-primary` : 見出し・主要な値（ダークグレー）
- `text-content-secondary` : ラベル・補足テキスト（ミディアムグレー）
- `text-content-muted` : 非活性・ヒント（ライトグレー）
- `bg-surface-bg` : ページ・カード背景（ほぼ白）
- `border-surface-border` : カード・パネルの枠線（ライトグレー）
- `bg-surface-muted` : ホバー・選択背景（極薄グレー）
- `bg-brand-default` : プライマリボタン・ブランドカラー（グリーン）
- `text-brand-default` : 選択状態・リンク（グリーン）
- `bg-alert-bg` / `text-alert-text` : アラート状態（薄赤 / 赤）

**栽培方式バッジ**
- 水耕（hydroponic）: 青系バッジ
- 土壌（soil）: 緑系バッジ
- 両対応（both）: 紫系バッジ

**アイコン（lucide-react）**
- 植物・栽培: `Leaf`
- 追加: `Plus`
- 設定: `Settings`
- 削除: `Trash2`
- 編集: `Pencil`

## データモデル

```typescript
// 植物マスタ
type Plant = {
  id: string;
  name: string;  // 例: "バジル", "ミニトマト", "サラダ菜"
  cultivation_type: "hydroponic" | "soil" | "both";
  created_by: string | null;
};

// センサー種別ごとの適正値・アラート閾値
type PlantThreshold = {
  plant_id: string;
  sensor_type_id: string;  // 例: "temperature", "humidity", "ec", "ph", "water_temp"
  optimal_min: number | null;
  optimal_max: number | null;
  alert_min: number | null;
  alert_max: number | null;
};

// センサー種別マスタ
type SensorType = {
  id: string;  // "ec", "ph", "water_temp", "temperature", "humidity"
  name: string;  // "EC値", "pH", "水温", "気温", "湿度"
  unit: string;  // "mS/cm", "", "℃", "℃", "%"
};
```

## 画面仕様

### メイン画面（/settings/plants）

**レイアウト**
- ページタイトル「植物マスタ」（`Leaf` アイコン付き）+ 右端に「植物を追加」ボタン（`Plus` アイコン）
- 植物カードをグリッド表示（2〜3列、レスポンシブ）
- 植物が0件の場合は中央に空状態メッセージ + 「植物を追加」ボタン

**植物カード（各植物ごと）**
- 植物名（`text-content-primary`、太字）
- 栽培方式バッジ（水耕 / 土壌 / 両対応）
- 登録済み閾値のサマリー（例:「EC: 1.5–2.0 mS/cm」「pH: 6.0–6.5」）。未設定のセンサーは表示しない
- フッター: 「閾値を編集」ボタン（`Settings` アイコン） + 「削除」アイコンボタン（`Trash2`）
- ホバーで薄いシャドウ

**サンプルデータ（デザイン用）**
- バジル（水耕）: EC 1.5–2.0 mS/cm / pH 6.0–6.5 / 水温 20–26℃
- ミニトマト（土壌）: 気温 18–28℃ / 湿度 60–80%
- サラダ菜（両対応）: 閾値未設定
- ルッコラ（水耕）: EC 1.2–1.8 mS/cm / pH 6.0–7.0

---

### モーダル A: 植物を追加

**トリガー**: 「植物を追加」ボタン

**フォーム**
- 植物名（テキスト入力、必須、例プレースホルダー: 「例: バジル」）
- 栽培方式（ラジオ or セグメントボタン: 水耕 / 土壌 / 両対応）
- 閾値の入力（オプション）: センサー種別ごとに4列（アラート下限 / 適正下限 / 適正上限 / アラート上限）。追加できるセンサー種別一覧から選んで行追加できるUI
- キャンセル・保存ボタン

---

### モーダル B: 閾値を編集

**トリガー**: カードの「閾値を編集」ボタン

**レイアウト**
- 上部: 植物名 + 栽培方式バッジ
- テーブル形式で各センサー種別の閾値を編集
  - 列: センサー種別 | アラート下限 | 適正下限 | 適正上限 | アラート上限 | 削除
  - 行: EC値、pH、水温 など（登録済みのもの）
  - 最下部に「+ センサーを追加」リンク
- バリデーション: アラート下限 ≤ 適正下限 ≤ 適正上限 ≤ アラート上限
- キャンセル・保存ボタン

**閾値の視覚ヒント**（オプション）
- 入力フィールドの下に「値の目安」をテキストで表示（例: バジルのEC推奨値 1.5〜2.0）

---

## デザインの重点

1. **農業・植物らしさ**: グリーン系のアクセントカラー、`Leaf` アイコンを随所に
2. **閾値編集UIのわかりやすさ**: アラート下限→適正下限→適正上限→アラート上限の順序関係が視覚的に明確（グラデーションバーや矢印で表現）
3. **空状態の親切な誘導**: 0件時に「まず植物を追加して、センサーの適正値を設定しましょう」
4. **一貫したshadcn/uiスタイル**: ゾーン管理画面やアラート画面と同じトーン

デスクトップ（1280px想定）でデザインしてください。
メイン画面、モーダルA（植物を追加）、モーダルB（閾値を編集）の3パターンを見せてください。
